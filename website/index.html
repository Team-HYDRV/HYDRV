Guide

How to use HYDRV.

A quick walk-through of the parts people actually use every day.

01

Allow permissions first

HYDRV needs internet access and install permission before it can download or install anything. Once that is done, you’re ready to go.

02

03

Keep Favorites close

Star the apps you use often so they’re easy to find again later.

04

Check Installed

See what’s already on your phone and check whether HYDRV has something newer.

05

Manage Downloads

Watch files finish, install APKs, retry failed downloads, or clear old items when you’re done with them.

06

Use General and About

General is where you change how HYDRV looks and behaves. About is where you’ll find version info, reports, and project links.

Backend tutorial

For power users who want their own source.

Most people can stay on Default. If you want HYDRV to point at your own catalogue, this is where it happens.

01

What it does

The backend lets HYDRV load catalogue data from your own JSON source instead of only the built-in one.

02

Why it helps

It’s useful if you want a private list, a test source, or a mirrored catalogue that you control yourself.

03

How to set it up

Open General, tap Backend, add or select a URL, and make sure it returns valid HYDRV catalogue data.

04

Switch back anytime

If you just want the normal HYDRV setup again, switch back to Default and keep using the built-in source.

If a backend is broken or offline, HYDRV may not be able to load apps from it. For most people, Default is still the safest choice.

FAQ

A few quick answers.

What is HYDRV?

HYDRV is the new name for the project that used to be xManager. It is still the same Android app for browsing releases and downloading files through signed links.

Is this the full public website?

Yes. This is the current public landing page, and future updates can expand it into a release archive, build details, and more project context.

Can this be deployed to hydrv.app?

Yes. It is a static site and can be deployed to Cloudflare Pages, Netlify, GitHub Pages, or any other static host.
