# Contributing

Thanks for helping with HYDRV.

## Before you open a pull request

1. Make your change in a feature branch.
2. Run the app build locally if your change touches Android code:
   - `.\HYDRV\gradlew.bat assembleDebug`
3. Keep commits focused and descriptive.
4. Use the issue and pull request templates when possible.

## Good PRs

- Small and focused
- Clear summary of what changed
- Screenshots for visual changes
- Notes for backend or release-flow changes

## Release notes

Update [`CHANGELOG.md`](CHANGELOG.md) when you prepare a tag for release.
